Run Concordion-NET Tests with NUnit

Similar to the Java version, Concordion-NET integrates with one of the most popular testing frameworks. Concordion-NET provides an NUnit-Addin that allows executing active specifications with any NUnit environment that supports NUnit-Addins.

Please note that at the time of writing the NUnit Test Adapter for Visual Studio needs an update to fully support addins. Nevertheless, the Concordion-NUnit-Addin works fine with the standalone NUnit runner. Thus, we are describing here how to run Concordion-NET tests with the standard NUnit GUI runner.

  1. First download and install NUnit.
  2. If you have not already done, download Concordion-NET.
  3. Take the Concordion.NUnit.dll and copy it to the addin directory of our NUnit installation (<NUnit-installation-path>/bin/addins). Please, make sure to use the latest version of this DLL, when you update Concordion-NET.
  4. Make sure that the follwing DLLs are referenced in your project. You find the Concordion.dll and the necessary OGNL.dll in the package you have downloaded in step 2. The nunit.framework.dll is part of the NUnit installation from step 1.
    • concordion.dll
    • nunit.framework.dll
    • OGNL.dll
  5. Add the attribute "[assembly: RequiredAddin("ConcordionNUnitAddin")]" to the AssemblyInfo.cs of your project. This indicates NUnit that the Concordion-NUnit-Addin is required to run your Concordion-NET tests.
  6. Make sure that you use .NET framework 4.0 or higher as target framework version in your Spec project that contains your Concordion-NET fixtures.
  7. Make sure that all your active specifications have the property "Build Action" set to "Embedded Resource" in our Visual Studio project.
  8. Build your project containing your executable specifications.
  9. Finally, execute your Concordion-NET tests like any "usual" NUnit tests with the runner of your choice.

You will see the typical green bar, when your tests pass. :-)

You can find the resulting report pages in the output directory that can be specified in configuration files of Concordion-NET. By default Concordion-NET places them into the temp directory (on Windows defined by the environment variable %Temp%, which by default points to C:\Users\<your-windows-user>\AppData\Local\Temp).

If you want to debug your Concordion-NET tests, you can do this in the same way as debugging NUnit tests.
